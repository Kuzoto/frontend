# 404 Page Design

**Date:** 2026-03-04
**Status:** Approved

## Problem

The router has no catch-all route. Unmatched URLs fall through to `RouteErrorBoundary`, which shows a generic error layout — not a purposeful 404 experience.

## Decision

**Option A — Dedicated `NotFoundPage` + `*` catch-all route.**

Create `src/pages/NotFoundPage.tsx` and add `{ path: '*', element: <NotFoundPage /> }` as a top-level route. The page is completely standalone (no AppLayout, no sidebar, no TopNav). `RouteErrorBoundary` remains unchanged for runtime errors.

## Design

### Layout

Fullscreen centered column (`min-h-screen flex flex-col items-center justify-center`). Standalone — no nav or sidebar chrome.

### Visual

- Large `Home` icon (Lucide) in a warm amber container: `bg-amber-500/15 text-amber-600` — matches the Notes feature card pattern on the dashboard.
- Subtle radial amber gradient blob behind the icon.
- `404` in `text-8xl font-bold text-amber-500/40` — large, warm, present but not aggressive.
- Heading: **"This room doesn't exist."** (`text-foreground`)
- Subtitle: **"You've wandered outside My Noook."** (`text-muted-foreground`)
- No animations; consistent with the rest of the app.

### Auth-aware CTA

Uses `useAuth()` hook:
- Authenticated → **"Go to Dashboard"** → `/dashboard`
- Unauthenticated → **"Go to Login"** → `/login`

Single button. No "go back" — the user navigated to an unknown URL so going back may not be meaningful.

### Dark mode

Inherits the existing `.dark` class mechanism. All color tokens (`text-foreground`, `text-muted-foreground`, `bg-background`) adapt automatically.

## Files Changed

| File | Change |
|---|---|
| `src/pages/NotFoundPage.tsx` | New — standalone 404 page component |
| `src/router/index.tsx` | Add `{ path: '*', element: <NotFoundPage /> }` at top level |
